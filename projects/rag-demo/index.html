Retrieval Strategy Details

HyDE Query Expansion
Generate a hypothetical answer document, then embed it. Bridges the phrasing gap between questions ("Who is X?") and corpus documents ("X is a professor of…").
Dense Retrieval (FAISS)
BGE-base-en-v1.5 embeddings in a FAISS Flat L2 index. Captures semantic similarity even for paraphrased queries. Returns k=40 candidates.
Sparse Retrieval (BM25)
BM25Okapi over tokenized chunks. Critical for exact-match queries: course codes like CS188, faculty names, URLs. Returns k=40 candidates.
RRF + Cross-Encoder
Reciprocal Rank Fusion merges the two ranked lists. Cross-encoder (ms-marco-MiniLM-L-6-v2) scores each candidate against the query; top 20 pass to the LLM.
Keyword Boosting
Regex detects person names and course codes in the query. Matching chunks receive +0.5 score bonus — low-cost, high-impact for structured lookups.
